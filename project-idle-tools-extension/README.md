# Project-Idle-tools

> Chrome 扩展 — 自动化工具集，面向 [idle.charsgame.com](https://idle.charsgame.com) 的多账号管理、循环采集/制造/交易、数据统计。

## 功能一览

### 📋 账号管理
- 账号分组管理（内置分组 / 自定义分组）
- 一键登录/切换账号
- 拖拽移动账号到其他分组
- 账号备注重命名
- 导出/导入账号配置
- 记住上次使用账号

### 🔄 循环功能
| 功能 | 说明 |
|------|------|
| 循环切号 | 选定账号组循环登录，每个账号停留可配置的秒数后切换到下一个 |
| 循环采集 | 选定账号 + 资源（伐木/采矿/狩猎/种植），登录后跳转到采集页停留指定时长 |
| 循环出售 | 登录 → 查询背包 → 按市场最高买价自动出售指定商品 |
| 循环购买 | 登录 → 以指定价格和数量创建买入订单 |
| 循环领取 | 登录 → 查询已成交的买入订单 → 领取物品 |

### 📊 检测功能
| 功能 | 说明 |
|------|------|
| 检测技能等级 | 遍历账号，查询全部技能等级（含总等级、金币），支持汇总视图和详情导出 |
| 检测背包 | 遍历账号，查询背包物品数量，跨账号汇总 |
| 检测仓库 | 遍历账号，查询仓库物品，支持单账号查看 |

### 💹 市场价格
- 一键查询 43 种常用商品的买卖盘口
- 显示最高买入价 + 最低卖出价 + 深度
- 价差颜色提示（倒挂红色预警）
- 支持复制为文本格式
- 数据持久化，关闭弹窗不丢失，随时更新

### 📊 数据统计
- 汇总技能等级、背包/仓库、市场价格三项数据
- 跨账号统计（平均等级、总金币、Top物品排行等）
- 支持跳转到详情弹窗
- 数据在 localStorage 中持久化

### 📋 操作日志窗口
- 实时显示操作日志（账号名 + 步骤 + 结果状态）
- 可拖拽 + 可调整大小
- 跨页面切换自动恢复
- 颜色标记：🟢 成功 🟡 警告 🔴 失败

### 🔧 其他工具
- API Token 提取（带复制功能）
- 手动查询任意物品的价格
- 添加/删除账号
- 面板显示/隐藏切换

## 安装

### 方法一：加载解压的扩展（开发/自用）
1. 下载本仓库代码并解压
2. 打开 Chrome → `chrome://extensions`
3. 开启右上角 **开发者模式**
4. 点击 **加载已解压的扩展程序** → 选择解压后的文件夹

### 方法二：打包为 .crx（分发）
1. 在 `chrome://extensions` 中点击 **打包扩展程序**
2. 选择扩展程序目录 → 生成 `.crx` 和 `.pem`
3. 将 `.crx` 分发给其他人安装

## 使用指南

### 首次使用
1. 打开 [idle.charsgame.com/auth](https://idle.charsgame.com/auth) 登录页
2. 页面右下角会出现 👤 浮动按钮
3. 默认账号列表为空，点击 ➕（浮动按钮）或面板底部「添加账号」自行添加
4. 添加完成后点击账号即可自动填充登录

### 账号配置格式
在 `accounts-config.js` 中添加内置账号（可选）：
```javascript
window.PI_ACCOUNTS = {
    '分组名': [
        { name: '备注名', email: 'your@email.com', password: 'your_password' },
    ],
};
```
也可以在浏览器中通过面板的「添加账号」对话框添加，数据保存在 localStorage 中。

## 项目结构

```
project-idle-tools-extension/
├── manifest.json          # Chrome Manifest V3 配置
├── accounts-config.js     # 内置账号配置（可选，本地添加）
├── shared-styles.js       # 共享 CSS 样式（自注入）
├── shared-utils.js        # 共享工具函数
├── items-database.js      # 统一物品数据库（429个物品）
├── content.js             # 主脚本（面板 + 循环功能）
├── manual-login.js        # 手动选择账号登录
├── cycle-buy.js           # 循环购买
├── cycle-inventory.js     # 检测背包
├── cycle-warehouse.js     # 检测仓库
├── cycle-skill-level.js   # 检测技能等级
├── popup.html             # 扩展弹出面板
└── popup.js               # 弹出面板逻辑
```

## 技术说明

- Chrome Manifest V3
- 所有数据/状态通过 `localStorage` 持久化
- API 通过 `fetch` + Bearer Token 调用
- 登录通过 DOM 操作自动填表
- 循环机制：登录 → 执行 → 清除认证 → 跳转登录页 → 下一个账号

## 注意事项

- 账号密码存储在扩展文件或浏览器 localStorage 中，请妥善保管
- 循环功能会频繁切换账号，不要在游戏活跃时段长时间运行
- 市场价格 API 有频率限制，查询间隔已加入 150ms 延迟
